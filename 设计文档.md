# 工作日志生成工具 — 设计文档

## 1. 项目概述

### 1.1 背景
洱海科技小院硕士生张航境需要每天撰写工作日志。当前流程：手动在 Word 中写作，格式为叙事散文风格，包含工作内容、个人感想和图片。需要开发一个 Web 工具来辅助生成日志内容。

**历史日志位置**：`D:\日常\日志\`
- 当前月份（6月）：14 篇（6.1-6.16，根目录下）
- 3月归档：`2026年3月/` — 31 篇（3.1-3.31，每天均有）
- 4月归档：`2026年4月/` — 30 篇（4.1-4.30，每天均有）
- 5月归档：`2026年5月/` — 31 篇（5.1-5.31，每天均有）
- **合计**：约 106+ 篇（2026.3.1 至今，每天一篇）

### 1.2 目标
开发一个**支持多设备运行**的响应式 Web 应用。核心诉求：

- **多设备使用**：台式机（完整版，有历史日志）、笔记本（出差版，云端浓缩背景）。项目文件夹直接拷贝到新电脑即可运行
- 用户输入当日工作要点
- 上传素材文件（图片、PPT、PDF、Word、录音纪要等）
- AI 自动从素材中提取关键信息
- 按历史日志模板生成叙事散文风格的日志**纯文本内容**
- 用户可在 Web 界面预览、编辑、复制文本，自行粘贴到 Word 完成排版

### 1.3 部署方案
- **部署平台**：优先免费/学生优惠方案，降低使用成本
- **手机访问**：响应式 Web 设计，无需安装 App，手机浏览器直接打开

### 1.4 多设备方案

**问题**：106 篇历史日志存储在台式机 `D:\日常\日志\`，出差时笔记本无法访问。

**方案**：将历史日志浓缩提炼，存储到云端（腾讯云 COS，50GB 免费），出差设备启动时自动拉取。

**浓缩流程**（台式机执行）：
```
106篇原始日志 (~50万字)
  → 按月份分 4 批，DeepSeek 提取每月摘要（每篇 500 字）
  → DeepSeek 二次提炼，生成总背景画像（~2000 字）
  → 存储为 history_summary.json，上传到腾讯云 COS
```

**出差设备降级模式**：

| 功能 | 完整版（台式机） | 出差版（笔记本） |
|------|-----------------|-----------------|
| 历史日志 100+ 篇 | ✅ 全量索引 | ❌ 无本地文件 |
| 历史浓缩背景 | ✅ | ✅ 从 COS 拉取 |
| 风格画像 | ✅ | ✅ 拷贝 style_profile.json |
| Few-shot（近一周） | ✅ 从本地读取 | ❌ 无（降级到仅风格画像+背景） |
| 素材解析 | ✅ | ✅ |

**同步方式**：项目文件整体拷贝到新电脑，`pip install -r requirements.txt` 即可运行。

### 1.5 使用场景
1. 用户在任意设备（台式机/笔记本/手机）打开浏览器访问 Web 界面
2. 输入当日日期、工作要点（1. 2. 3. 等）
3. 可选上传素材文件（会议照片、PPT、PDF、录音纪要等）
4. 点击"生成日志"
5. AI 根据素材和历史日志风格生成完整日志文本
6. 用户预览、编辑、复制文本

## 2. 日志模板分析

### 2.1 日志结构（基于历史日志提炼）

| 板块 | 内容 | 是否必须 |
|------|------|----------|
| 标题 | `工作日志` | 是 |
| 日期引入 | "今天是X月X日。…" 自然引入 | 是 |
| 正文 | 按主题分段叙事，描述当天工作内容和思考 | 是 |
| 【个人感想】| 当天的收获、成长感悟 | 是 |
| 署名 | `—云南大学 张航境` | 是 |

### 2.2 写作风格特征
- **叙事散文体**：不是清单式"今日完成"，而是连贯的故事叙述
- **个人视角**：以"我"的视角，融入观察、感受和反思
- **深度思考**：不仅记录"发生了什么"，更要写"从中读懂了什么"
- **生动细节**：引用具体人物话语、数字、场景
- **主题串联**：将当天多件事用一条主线串联起来
- **学术+实践语气**：既有硕士生的学术思维，又有田间地头的真实感

### 2.3 命名与存储规则
- 日志目录：`D:\日常\日志\`
- 当月日志：`D:\日常\日志\YYYY.M.D-张航境-工作日志.docx`
- 归档目录：`D:\日常\日志\YYYY年M月\`
- 归档方式：**用户自行手动归档**（月底或下月初将当月日志移入归档目录）

## 5. 功能需求

### 5.1 核心功能

#### F1: 工作要点输入
- 文本输入框，支持多行输入
- 用户可以写"1. xxx 2. xxx 3. xxx"格式
- 也可以直接描述当天发生的事情

#### F2: 素材上传
- 支持文件类型：.jpg/.png/.pdf/.pptx/.docx/.xlsx/.txt/.m4a/.mp3/.wav
- 多文件上传，**数量无限制**
- 上传后显示文件列表，可逐个删除
- **★ 语音转文字（核心功能）**：.m4a/.mp3 等音频文件上传后，自动调用云端 ASR API 转为文字

#### F3: AI 素材解析
- 图片：调用 DeepSeek 视觉 API，提取图片中的内容描述
- PPT：用 python-pptx 提取文字
- PDF：用 PyPDF2 提取文字
- Word：用 python-docx 提取文字
- Excel：用 openpyxl 提取表格内容
- 音频（.m4a/.mp3/.wav）：**★ 核心功能**，调用云端 ASR API 转文字

#### F4: AI 日志生成
- 将用户要点 + 素材解析结果 + 历史日志风格（few-shot prompt）发给 DeepSeek
- 生成符合模板的完整日志（标题 + 日期引入 + 正文 + 个人感想 + 署名）
- 流式输出（打字机效果），用户实时看到生成过程

#### F5: 日志预览与编辑
- 纯文本预览（AI 生成的原始文本）
- 一键切换编辑模式（textarea），自由修改
- 编辑/预览模式可随时切换
- 一键复制全文

### 5.2 辅助功能
- **配置页**：设置 DeepSeek API Key、日志目录路径等
- **历史日志列表**：查看本月已生成的日志

## 6. Prompt 设计（核心）

### 6.1 系统 Prompt 结构

```
你是一位在云南洱海科技小院工作的硕士生，名叫张航境。
你每天撰写工作日志，风格如下：

【风格要求】
- 叙事散文体，以"我"的第一人称视角
- 不仅记录"做了什么"，更要写"从中读懂了什么"
- 语言要有温度，有细节，有反思
- 要把当天多件事用一条主线串联起来
- 引用具体的人物、对话、数据、场景

【日志结构】
1. 标题：工作日志
2. 日期引入：今天是X月X日。...
3. 正文：按主题分段
4. 【个人感想】：当日成长感悟
5. 署名：—云南大学 张航境

【参考风格】（附 few-shot 历史日志）

【今日素材】（用户输入 + 素材解析结果）

请根据以上素材，生成今天的工作日志。
```

### 6.2 Few-shot 策略

#### 样本库
- 从 2026.3.1 至今，已有 **100+ 篇**历史日志
- 所有历史日志均可作为风格参考样本

#### 默认策略
- 每次生成日志时，从历史日志中选取**最近一周（7篇）**作为 few-shot 示例
- 目的：保持当前工作阶段的**语境连续性**
- 如果单篇太长导致 token 超限，优先保留最近 3-5 篇

#### 风格持续更新
- **每周更新一次写作风格画像**（自动）
- 实现方式：
  1. 每周选取该周全部日志（约 7 篇）
  2. 调用 DeepSeek 进行风格分析
  3. 将风格画像存储为 `style_profile.json`
  4. 下次生成日志时注入 Prompt
- 触发：每周一自动执行，也可手动触发

#### 地点感知
- **每次生成日志前**，AI 自动分析最近一周日志中的工作地点：
  - **基地阶段**：在弥渡/基地大棚做实验、育苗、田间管理
  - **小院阶段**：在古生村科技小院、考察接待、会议、文献研究
  - **其他地点**：出差、培训等
- 地点判断结果写入 Prompt

#### Few-shot 选择策略
```
默认：最近一周（7篇），保持阶段语境连续性和地点一致性
降级：最近3-5篇（token 超限时）
特殊：当前阶段无历史日志时，仅依赖风格画像 + 地点推断
```

#### Few-shot Token 管理

DeepSeek 上下文窗口约 64K tokens。每篇历史日志约 1000-2000 字。为防止超限：

| 场景 | 策略 |
|------|------|
| 7 篇总长度 < 50% 上下文窗口 | 全量加载 7 篇 |
| 7 篇总长度超过阈值 | 自动截断每篇至前 1500 字，仍超则减少至 3-5 篇 |
| 3 篇仍超 | 仅加载 1 篇最近的 + 风格画像文本描述 |
| 极端情况 | 仅使用 System Prompt + 风格画像，不加载原文 |

         │
         ▼

         │
         ▼

### 10.2 模块清单

#### 模块 1: 素材解析引擎 (`parser_engine/`) ★ 可复用
| 子模块 | 文件 | 功能 |
|--------|------|------|
| 文档解析 | `doc_parser.py` | 解析 .docx/.pdf/.pptx/.txt |
| 图片识别 | `image_parser.py` | 调用 DeepSeek 视觉 API 描述图片 |
| 语音转文字 | `audio_parser.py` | 调用云端 ASR API 转文字 |

**统一接口**：`parse(file_path, **options) -> str`

#### 模块 2: AI 生成引擎 (`generator_engine/`) ★ 可复用
| 子模块 | 文件 | 功能 |
|--------|------|------|
| Prompt 构建 | `prompt_builder.py` | 拼接系统 Prompt + 风格画像 + few-shot + 素材 |
| LLM 调用 | `llm_client.py` | 封装 DeepSeek API，流式输出 |
| 风格画像 | `style_profile.py` | 每周分析历史日志，更新写作风格描述 |

**核心流程**：`build_messages(...)` → `chat_stream(...)` 流式返回文本

#### 模块 3: 历史日志引擎 (`history_engine/`)
| 子模块 | 文件 | 功能 |
|--------|------|------|
| 日志索引 | `indexer.py` | 扫描目录，建立日期→路径映射 |
| 日志读取 | `reader.py` | 读取 .docx 日志内容，返回纯文本 |
| Few-shot 选取 | `selector.py` | 按策略选取 few-shot（四级 token 降级） |

**统一接口**：`get_recent_logs(date, count) -> list[str]`

#### 模块 4: 历史浓缩引擎 (`summary_engine/`)
将 100+ 篇历史日志压缩为可跨设备共享的精炼背景。

| 子模块 | 文件 | 功能 |
|--------|------|------|
| 月度摘要 | `monthly_summary.py` | 按月分批提炼每月摘要 |
| 总背景画像 | `background.py` | 二次提炼生成总背景画像 |
| 云端同步 | `shared/cos_utils.py` | 上传/下载到腾讯云 COS（50GB 免费） |

#### 模块 5: Web 层
| 组件 | 文件 | 功能 |
|------|------|------|
| Flask 入口 | `app.py` | 路由定义、请求调度 |
| 前端页面 | `templates/index.html` | 单页应用 |
| 前端样式 | `static/style.css` | UI 样式 |
| 前端逻辑 | `static/app.js` | 文件上传、流式渲染、编辑、复制 |

## 11. 错误码与安全性

### 11.1 错误码规范

| HTTP 状态码 | 场景 | 响应格式 |
|-------------|------|----------|
| 200 | 成功 | `{"ok": true, ...}` |
| 400 | 参数缺失/格式错误 | `{"error": "描述"}` |
| 500 | 服务器内部错误 | `{"error": "描述"}` |

### 11.2 安全设计

| 项 | 措施 |
|----|------|
| **API Key 存储** | 存储于 config.json。部署后通过环境变量注入 |
| **文件上传** | 白名单限制类型，单文件最大 50MB |
| **路径穿越** | 文件名使用 UUID 前缀 |
| **敏感文件** | `.gitignore` 排除 config.json、uploads/ |
| **XSS** | 前端使用 textContent，避免注入 |

## 12. 已确认事项

| 序号 | 事项 | 决定 | 影响 |
|------|------|------|------|
| 1 | 输出格式 | 只生成纯文本内容，用户自行粘贴到 Word 排版 | 砍掉 exporter_engine，简化架构 |
| 2 | 图片视觉识别 | 先用 DeepSeek，体验不好再换 | Phase 3 直接用 DeepSeek 视觉 API |
| 3 | 录音文件(.m4a) | 上传后由云端 ASR API 转为文字 | ★ 核心功能 |
| 4 | 多设备使用 | 项目文件夹拷贝到新电脑即可运行。出差设备从 COS 拉取历史浓缩背景 | 台式机（完整版）vs 笔记本（浓缩版）自动降级 |
| 5 | 部署平台 | 优先免费方案（Render/Railway），备选学生服务器 | Phase 9 调研 |
| 6 | 历史日志浓缩 | 106 篇分层压缩为月度摘要 + 总背景，存到腾讯云 COS | Phase 10 |
| 7 | 归档 | 用户自行手动归档 | 无需自动归档功能 |
| 8 | 风格更新 | 每周自动分析近一周日志，更新 style_profile.json | AI 写作风格持续跟随用户演进 |

| 阶段 | 内容 | 涉及模块 | 预计产出 |

## 14. 风险与应对

| 风险 | 应对 |
|------|------|
| DeepSeek 不支持图片视觉识别 | 单独接入通义千问免费视觉 API |
| AI 生成风格与历史日志不一致 | Few-shot + 每周风格画像逐步逼近 |
| 运行环境不稳定 | requirements.txt 锁定依赖，venv 隔离 |
| 云端路径失效 | 通过环境变量或配置页设置 logs_dir |
| Few-shot 超出 token 限制 | 四级降级策略（详见 §6.2） |
